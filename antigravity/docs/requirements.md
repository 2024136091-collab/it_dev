# 기획서 및 요구사항

## 프로젝트 배경

대한민국에는 약 80만 대 이상의 승강기가 설치·운행되고 있으며, 안전한 운행을 위해 정기 점검이 법적으로 의무화되어 있습니다. 그러나 승강기 설치·점검 정보는 국가승강기정보센터 PC 웹에서만 조회 가능해 모바일 접근성이 떨어지고, 건물명이나 승강기 번호로 빠르게 검색하기 어렵습니다.

## 비전

> 승강기 안전 정보를 누구나 손 안에서 빠르게 확인할 수 있는 세상

## 목표

국가승강기정보센터 공공 API 기반의 Flutter 앱으로 승강기 정보를 모바일에서 쉽게 조회한다.

## 기능 요구사항

| ID | 요구사항 | 우선순위 |
|---|---|---|
| F-01 | 건물명·주소·승강기 번호로 검색 | 높음 |
| F-02 | 지역 필터 (8개 권역) | 높음 |
| F-03 | 건물 유형 필터 (학교·병원·아파트 등) | 중간 |
| F-04 | 승강기 종류 필터 | 중간 |
| F-05 | 건물별 그룹핑 표시 | 중간 |
| F-06 | 승강기 상세 정보 조회 | 높음 |
| F-07 | 점검 이력 조회 | 높음 |
| F-08 | 즐겨찾기 저장·해제 | 중간 |
| F-09 | 스마트 자연어 검색 (규칙 기반) | 낮음 |
| F-10 | 위치 지도 조회 (OpenStreetMap) | 낮음 |

## 비기능 요구사항

| 항목 | 내용 |
|---|---|
| 플랫폼 | Flutter Web (Android·iOS 추후 지원) |
| 상태 관리 | Riverpod (StateNotifier + AsyncValue) |
| 아키텍처 | 4레이어 (Presentation·Application·Domain·Data) |
| 데이터 | 국가승강기정보센터 API 구조 기반 더미 데이터 |
